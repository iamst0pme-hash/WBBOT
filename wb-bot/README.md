# WB Sales History Bot

Telegram-бот для выгрузки Excel по продажам через метод Wildberries:

- `POST /api/analytics/v3/sales-funnel/products/history`

Бот делает:
1. Показывает кнопку **Продажи**
2. Даёт выбрать период
3. Забирает все карточки товаров
4. Получает историю продаж по `nmIds` чанками по 20
5. Забирает текущие остатки WB
6. Формирует `.xlsx`
7. Отправляет файл в Telegram

## Что внутри Excel

Лист `Продажи` содержит столбцы:

- Артикул Продавца
- Заказы, шт
- Заказы, сумма
- Остатки, шт
- Динамика заказов в рублях

> Динамика = сумма заказов за выбранный период минус сумма заказов за сопоставимый предыдущий период такой же длины.

## Важное ограничение WB API

Метод `sales-funnel/products/history` по официальной документации позволяет получать данные **максимум за последнюю неделю** и требует `nmIds` массивом от **1 до 20** элементов. Лимит — **3 запроса в минуту** на аккаунт. Поэтому в боте доступны только безопасные диапазоны, укладывающиеся в ограничения API.

## Доступные периоды

- Вчера
- Последние 7 дней
- Прошлая неделя

## Переменные окружения

- `BOT_TOKEN` — токен Telegram-бота
- `WB_ANALYTICS_TOKEN` — токен WB для analytics
- `WB_CONTENT_TOKEN` — токен WB для content

Если у тебя один токен покрывает обе категории, можно использовать одинаковое значение в обеих переменных.

## Запуск локально

```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
python -m app.main
```

## Railway

1. Создай проект из GitHub-репозитория
2. Добавь переменные из `.env.example`
3. Railway запустит команду из `Procfile` / `railway.toml`

## Почему отчёт может собираться долго

Wildberries ограничивает history-запросы до 3 в минуту. Если у тебя много товаров, бот будет честно идти чанками по 20 `nmIds` и соблюдать rate limit, чтобы не ловить 429.
